# Privacidade e uso de dados - rascunho

## Principio

Este repositorio nao deve conter dados sensiveis, segredos, credenciais, tokens, dados pessoais desnecessarios, dados de clientes, dados de times ou informacoes confidenciais que nao sejam indispensaveis ao objetivo do projeto.

## Dados proibidos sem justificativa e protecao

- senhas;
- tokens;
- chaves de API;
- credenciais;
- dados pessoais sensiveis;
- dados medicos;
- dados financeiros privados;
- documentos contratuais confidenciais;
- informacoes internas de clientes;
- nomes reais em exemplos sem necessidade;
- dumps de sistemas corporativos;
- outputs de IA com informacao sensivel.

## Uso de exemplos

Dados de exemplo devem ser sinteticos, anonimizados ou sanitizados. Quando dados reais forem usados em teste, eles nao devem ser commitados sem autorizacao e revisao.

## Uso com IA

Arquivos deste repositorio nao devem ser enviados a ferramentas de IA externas, ambientes publicos ou modelos de terceiros sem avaliar risco de privacidade, confidencialidade e propriedade intelectual.

## Separacao de contexto

Contexto privado de Richard, diario pessoal, arquitetura administrativa, decisoes internas e material sensivel do ecossistema ShadeVanSoul nao devem ser misturados em pacotes de aplicacao, skills publicas ou versoes comerciais.
